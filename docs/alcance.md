# Alcance inicial

Este documento define qué se incluye y qué queda fuera del desarrollo inicial de AuraNails.

El objetivo es evitar que el proyecto crezca sin control antes de tener una primera versión realmente usable.

## Alcance de la primera etapa

La primera etapa debe permitir que una manicura pueda gestionar lo básico de su trabajo diario.

Incluye:

- Registrar clientas.
- Registrar servicios.
- Crear y consultar turnos.
- Cambiar estado de un turno.
- Registrar cobros.
- Registrar señas.
- Consultar un resumen simple del día y del mes.
- Registrar productos básicos.
- Ver productos con bajo stock.
- Registrar gastos simples.

## MVP funcional

El MVP debe responder estas preguntas:

- ¿Qué turnos tengo hoy?
- ¿Quién es mi próxima clienta?
- ¿Qué servicio se realiza?
- ¿Cuánto tengo que cobrar?
- ¿Ya dejó seña?
- ¿Cuánto cobré este mes?
- ¿Qué productos se están terminando?

## Fuera de alcance inicial

No se desarrollará al inicio:

- Facturación fiscal.
- Integración real con WhatsApp Business API.
- Integración real con Mercado Pago.
- App nativa Android/iOS.
- Multi-sucursal.
- Gestión contable avanzada.
- Reportes complejos.
- Control avanzado de empleados.
- Chatbot automático.
- Publicación automática en Instagram o TikTok.

Estas funciones pueden analizarse en etapas futuras.

## Criterio para agregar funciones

Antes de agregar una función nueva, debe responderse:

1. ¿Resuelve un problema real del trabajo diario?
2. ¿La usuaria lo entendería sin explicación técnica?
3. ¿Aporta valor al MVP o puede esperar?
4. ¿Complica demasiado el flujo actual?

Si la respuesta no es clara, la función queda pendiente.

## Prioridad actual

La prioridad es construir una app simple y usable.

Orden recomendado:

1. Clientas.
2. Servicios.
3. Turnos.
4. Cobros y señas.
5. Dashboard básico.
6. Productos.
7. Gastos.
8. Fotos de trabajos.
