## Riesgos detectados

### Riesgo 1: Sobrecargar la app

AuraNails puede crecer demasiado rápido si se agregan muchas funciones desde el inicio.

Mitigación:

- Respetar el MVP.
- Desarrollar por capas.
- Evitar funciones decorativas.

### Riesgo 2: Usar lenguaje demasiado técnico

La usuaria final puede tener poco o nulo conocimiento de sistemas.

Mitigación:

- Usar nombres simples.
- Priorizar botones claros.
- Evitar términos administrativos complejos.

### Riesgo 3: Integraciones prematuras

WhatsApp, Mercado Pago y almacenamiento externo aportan valor, pero pueden complicar el inicio.

Mitigación:

- Dejar integraciones para capas futuras.
- Primero construir el flujo manual completo.

## Decisión actual

Primero se construye una app simple y funcional.

Luego se escala.
